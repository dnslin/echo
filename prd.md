## 2. 产品背景

现在市面上的语音软件很多，但对一部分 PC 游戏玩家来说存在明显问题：

1. 软件太重，启动慢，占用高。
2. 功能太多，但真正开黑只需要“进房间、能说话、能听见”。
3. 创建频道、拉朋友、加入语音的步骤偏繁琐。
4. 一些软件对游戏场景不够友好，比如按键说话不稳定、窗口打扰、后台体验差。
5. 朋友之间临时开黑，不一定想注册账号、加好友、建服务器、进复杂社区。

本产品希望提供一个更轻、更直接、更适合朋友之间 PC 游戏语音交流的工具。

| 痛点 | 表现 | 产品应对方向 |

## 6. MVP 产品目标

MVP 只追求一个目标：

> 让一群朋友在 Windows 电脑上，可以快速进入同一个临时语音房间，并稳定完成游戏语音交流。

### 6.1 核心目标

| 目标 | 描述 |
| --- | --- |
| 快速进入 | 用户打开软件后，可以在极少步骤内创建或加入语音房间 |
| 无需注册 | 用户不需要账号、手机号、好友关系，只需本地昵称 |
| 临时房间 | MVP 只支持一次开黑会话使用的临时房间 |
| 语音稳定 | 说话、听声、静音、重连、恢复说话都要直观可靠 |
| 适合游戏 | 支持后台按键说话、自由说话、系统托盘、低打扰提示 |
| 状态清楚 | 用户一眼知道自己能否被听见、是否断线、谁正在说话 |

### 6.2 MVP 成功条件

产品 MVP 可以认为达标的条件：

```text
1. 首次用户可以设置昵称和随机头像。
2. 用户可以快速创建临时房间并获得邀请码。
3. 用户可以通过 6 位邀请码加入临时房间。
4. 两名用户可以进入同一房间并语音。
5. 3-10 名用户可以在同一房间进行语音交流。
6. 第 11 名用户无法加入，并看到明确提示。
7. 用户可以按住指定快捷键说话，松开后停止说话。
8. 应用在后台、游戏在前台时，按键说话仍应可用。
9. 用户可以切换为自由说话，但不会默认自动开麦。
10. 用户可以静音和取消静音自己。
11. 用户可以看到房间成员、谁正在说话、谁静音、谁重连中。
12. 用户可以选择麦克风设备和输出设备。
13. 用户可以测试麦克风输入，并调节整体房间语音音量。
14. 用户关闭窗口后，应用最小化到系统托盘，语音不中断。
15. 网络异常时有重连状态、失败提示和可操作按钮。
16. Windows 10 / Windows 11 x64 下可以正常安装、启动和使用。
```

## 8. 核心概念与产品边界

### 8.1 临时房间

MVP 只支持临时房间。

定义：

```text
临时房间是一次朋友开黑会话使用的语音房间。
用户通过邀请码邀请朋友加入。
临时房间不承诺长期保留，也不承诺第二天仍可用。
```

生命周期规则：

```text
有成员在线时，房间保持有效。
最后一名成员离开后，房间保留 30 分钟。
30 分钟内有人通过原邀请码加入，则房间恢复有效。
超过 30 分钟无人加入，则房间失效，邀请码不可再用。
```

创建成功但无人进入时：

```text
房间创建成功后，邀请码立即有效。
如果 30 分钟内没有任何成员进入，房间失效。
```

### 8.2 固定房间

固定房间不属于 MVP。

```text
固定房间
长期邀请码
上次房间自动恢复
固定小队房间
房间历史
```

以上均放到后续版本考虑。

### 8.3 本地匿名身份

MVP 不做账号系统。

```text
用户不需要注册账号。
用户不需要手机号。
用户不需要好友关系。
应用在本机生成匿名身份，用于识别本机用户和支持重连。
昵称只是展示名，不是唯一身份。
同一房间允许重复昵称。
```

### 8.4 房主

MVP 中，房主只是创建者标识。

```text
创建临时房间的用户显示为房主。
房主没有额外管理权。
房主不能踢人。
房主不能撤销邀请码。
房主不能转让房主。
房主不能关闭房间。
```

房主管理能力作为后续待办。

## 10. 用户流程

### 10.1 首次启动流程

```text
用户打开应用
        ↓
本地没有昵称
        ↓
显示昵称页
        ↓
用户输入昵称
        ↓
生成随机头像
        ↓
进入首页
```

首次昵称页包含：

```text
昵称输入框
继续按钮
简短说明：无需注册，昵称仅用于房间内显示
轻量隐私说明入口或简短说明
```

再次启动：

```text
本地已有昵称 → 直接进入首页
```

### 10.2 首页流程

首页包含：

```text
当前昵称 / 随机头像
快速创建房间
输入邀请码加入
自定义房间名入口
设置入口
```

首页不包含：

```text
最近房间
上次房间
自动加入
固定小队入口
社区入口
好友入口
```

### 10.3 快速创建房间流程

```text
用户点击“快速创建房间”
        ↓
系统使用默认房间名：{昵称}的房间
        ↓
创建临时房间
        ↓
生成邀请码
        ↓
进入创建成功页
```

### 10.4 自定义房间名创建流程

```text
用户点击“自定义房间名”
        ↓
进入创建房间页
        ↓
输入房间名称，可选
        ↓
点击创建
        ↓
创建临时房间
        ↓
生成邀请码
        ↓
进入创建成功页
```

### 10.5 创建成功页流程

创建成功页包含：

```text
房间名称
邀请码
复制邀请码按钮
进入房间按钮
返回首页 / 取消
```

创建成功示例：

```text
房间：今晚打瓦
邀请码：K7M9Q2
[复制邀请码] [进入房间]
```

规则：

```text
创建成功后，邀请码立即有效。
房主点击“进入房间”前，不在成员列表中，不参与语音。
其他人可以在房主进入前通过邀请码加入。
```

### 10.6 加入房间流程

```text
用户点击“输入邀请码加入”
        ↓
输入朋友发来的邀请码
        ↓
点击加入
        ↓
系统规范化邀请码并校验
        ↓
校验成功后直接进入语音房间页
        ↓
房间页显示语音连接状态
```

加入成功后不做加入成功页，也不做房间预览确认。

### 10.7 房间内语音流程

```text
用户进入房间页
        ↓
显示语音连接状态：连接中
        ↓
连接成功后显示：已连接
        ↓
默认语音模式：按键说话
        ↓
默认麦克风不持续发送
        ↓
按住快捷键发送语音
        ↓
松开快捷键停止发送
```

用户可在房间内切换：

```text
按键说话
自由说话
自己静音 / 取消静音
```

## 11. 数据与输入规则

### 11.1 昵称规则

```text
昵称长度：1-16 个字符
保存前自动去除前后空格
不允许空昵称
不允许纯空格
允许中文、英文、数字、空格和常见符号
不要求全局唯一
同房间允许重复昵称
```

错误提示：

| 场景 | 文案 |
| --- | --- |
| 昵称为空 | 请输入昵称 |
| 昵称太长 | 昵称最多 16 个字符 |

### 11.2 随机头像规则

MVP 支持随机头像。

```text
首次设置昵称后，为本机匿名身份生成随机头像。
随机头像保存在本地设置中。
用户可以在设置页重新随机头像。
用户不能上传头像。
用户不能从头像库选择头像。
用户不能编辑或裁剪头像。
```

头像形式不在 PRD 中固定，可以是：

```text
随机颜色圆形头像
默认图案头像
首字母头像
```

房间内修改头像：

```text
用户重新随机头像后，本机立即更新。
如果当前在房间内，成员列表头像同步更新给其他成员。
不显示“更换头像”系统消息。
不保留历史头像。
```

### 11.3 房间名称规则

```text
房间名称可选。
保存前自动去除前后空格。
未填写或纯空格时，使用 {昵称}的房间。
填写时长度为 1-24 个字符。
允许中文、英文、数字、空格和常见符号。
不要求唯一。
```

错误提示：

```text
房间名称最多 24 个字符
```

### 11.4 邀请码规则

```text
邀请码为 6 位大写字母数字组合。
示例：K7M9Q2
```

用户输入邀请码时，提交加入时进行规范化：

```text
去除前后空格
去除中间空格
去除短横线 -
转成大写
验证是否为 6 位字母数字
```

以下输入应识别为同一个邀请码：

```text
K7M9Q2
k7m9q2
K7M 9Q2
K7M-9Q2
```

错误提示：

```text
邀请码应为 6 位字母或数字
```

### 11.5 房间人数规则

```text
MVP 单房间最多 10 名在线成员。
该上限固定，不支持用户配置。
当房间已有 10 名在线成员时，第 11 名用户加入失败。
```

失败提示：

```text
房间人数已满，暂时无法加入。
```

## 12. 功能需求

## 12.1 昵称与本地身份

### 功能描述

用户不需要注册账号，但首次使用必须设置昵称。应用在本机生成匿名身份，用于房间展示、状态同步和重连。

### 页面与行为

```text
首次启动必须设置昵称。
昵称保存到本地。
再次启动直接进入首页。
用户可在设置页修改昵称。
同房间允许重复昵称。
当前用户在成员列表中应有“我”标识。
```

### 房间内改昵称

```text
用户可在设置页修改昵称。
本机立即保存。
如果当前在房间内，成员列表昵称实时同步。
其他成员看到更新后的昵称。
不显示改名系统消息。
不保留历史昵称。
```

### 优先级

P0。

### 验收标准

```text
首次打开应用时，必须设置昵称。
昵称为空时不能继续。
昵称会显示在房间成员列表中。
用户可以在设置页修改昵称。
房间内修改昵称后，其他成员能看到更新。
同一房间可以出现重复昵称。
```

## 12.3 创建房间

### 功能描述

用户可以创建一个临时语音房间，并获得邀请码。

### 创建方式

```text
快速创建房间
自定义房间名创建
```

### 房间信息

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 房间名称 | 否 | 不填时使用 {昵称}的房间 |
| 邀请码 | 自动生成 | 6 位大写字母数字组合 |
| 房主 | 自动设定 | 创建者为房主，仅作为标识 |
| 房间人数上限 | 固定 | MVP 固定 10 人 |

### 创建成功页

创建成功后进入创建成功页，不直接进入语音房间。

创建成功页包含：

```text
房间名称
邀请码

进入房间按钮
返回首页 / 取消
```

### 邀请码复制反馈

点击复制邀请码后：

```text
复制到剪贴板
按钮短暂显示“已复制”
可选显示应用内 Toast：“邀请码已复制”
不弹 Windows 桌面通知
```

复制失败时提示：

```text
复制失败，请手动复制邀请码
```

### 优先级

P0。

### 验收标准

```text
用户可以快速创建临时房间。
用户可以自定义房间名创建临时房间。
创建成功后看到创建成功页。
创建成功页展示房间名和邀请码。
用户可以一键复制邀请码。
用户点击进入房间后进入语音房间页。
创建成功后邀请码立即有效。
```

## 12.4 加入房间

### 功能描述

用户通过邀请码加入朋友的临时语音房间。

### 输入规则

```text
提交时规范化邀请码。
规范化后必须是 6 位字母数字。
大小写不敏感。
忽略空格和短横线。
```

### 加入失败情况

| 情况 | 提示文案 | 操作 |
| --- | --- | --- |
| 邀请码格式错误 | 邀请码应为 6 位字母或数字 | 重新输入 |
| 邀请码不存在 | 邀请码无效，请检查后重试 | 重新输入 / 返回首页 |
| 房间已过期 | 该房间已过期，请让朋友重新创建 | 返回首页 / 创建新房间 |
| 房间人数已满 | 房间人数已满，暂时无法加入 | 返回首页 / 重试 |
| 连接失败 | 连接失败，请检查网络后重试 | 重试 / 返回首页 / 反馈问题 |

MVP 不包含“被房主移出”失败状态，因为 MVP 不支持房主踢人。

### 优先级

P0。

### 验收标准

```text
用户输入有效邀请码后直接进入房间页。
输入无效邀请码时有明确提示。
加入失败时不应导致应用卡死或退出。
阻塞性错误必须提供下一步操作。
```

## 12.5 语音房间页

### 功能描述

语音房间页是产品核心页面。用户在这里查看房间状态、成员状态、自己的语音状态，并进行语音控制。

### 信息优先级

最高优先级：

```text
1. 自己当前语音状态
2. 谁正在说话
3. 连接状态
```

次优先级：

```text
4. 当前按键
5. 成员列表
6. 房间名和邀请码
7. 复制邀请
8. 离开房间
9. 设置入口
```

房间页必须让用户一眼知道：

```text
我现在有没有在发送声音？
朋友能不能听见我？
我能不能听见朋友？
当前谁在说话？
我是不是断线了？
```

### 页面内容

```text
房间名称
邀请码
复制邀请按钮
连接状态
成员列表
自己的语音状态
语音模式切换
自己静音 / 取消静音
当前按键
按键状态
简化输入反馈
设置入口
离开房间按钮
```

### 成员列表展示

每个成员应展示：

```text
随机头像
昵称
是否为“我”
是否为房主
是否正在说话
是否自己静音
是否重连中 / 连接异常
```

成员列表不展示：

```text
他人的麦克风设备异常
他人的输出设备异常
他人的本地诊断信息
```

### 正在说话定义

```text
正在说话 = 检测到有效语音输入。
不是单纯麦克风发送通道打开。
```

规则：

```text
按住快捷键但没有声音输入时，不一定对其他成员显示“正在说话”。
自由说话开启但环境安静时，不一定对其他成员显示“正在说话”。
其他成员只在检测到有效输入时显示正在说话。
```

### 自己的输入反馈

房间页应显示简化输入反馈：

```text
当前是否检测到我的声音
当前是否正在说话
```

按键说话模式下，房间页应显示：

```text
当前快捷键
按键状态：未按下 / 已按下
输入状态：未检测到声音 / 正在说话
```

### 优先级

P0。

### 验收标准

```text
用户可以看到当前房间成员。
用户可以看到谁正在说话。
用户可以看到自己是否静音。
用户可以看到自己是否连接中或重连中。
用户可以看到自己的按键状态。
用户可以看到自己的简化输入反馈。
用户可以复制邀请码。
用户可以离开房间。
用户可以进入设置页。
```

## 12.6 按键说话

### 功能描述

用户按住指定快捷键时，麦克风发送声音；松开快捷键后，停止发送声音。

这是本产品的核心功能之一。

### 默认行为

```text
默认语音模式：按键说话
默认快捷键：V
按下 V：满足发送条件
松开 V：停止发送
```

用户首次进入房间时：

```text
默认处于按键说话模式。
麦克风不持续发送。
按住默认快捷键 V 才发送语音。
```

### 快捷键设置

MVP 支持修改按键说话快捷键。

必须支持：

```text
键盘单键
鼠标侧键 4
鼠标侧键 5
```

不重点承诺：

```text
组合键
```

不承诺：

```text
鼠标宏键
手柄按键
多键复杂组合
游戏内虚拟按键
```

### 快捷键允许与禁止

允许设置：

```text
字母键、数字键
F1-F12
CapsLock
Tab
Shift / Ctrl / Alt 单键
空格
鼠标侧键 4/5
```

禁止设置：

```text
Esc
Win
Alt+F4
鼠标左键
鼠标右键
Enter
```

非法按键提示：

```text
该按键不能用于按键说话，请换一个按键。
```

### 后台游戏中可用

MVP 必须支持：

```text
当语音应用不在前台，且用户正在全屏、无边框窗口或窗口化游戏中时，按键说话快捷键仍应被识别。
```

兼容性边界：

```text
MVP 不承诺所有带反作弊系统的游戏都 100% 可识别。
MVP 不承诺管理员权限游戏下 100% 可识别。
MVP 不承诺远程桌面或虚拟机里的按键一定可用。
```

### 兼容性提示

按键设置或测试区域附近应提供轻量提示：

```text
如果快捷键在游戏中无效，请尝试：
- 更换为其他按键
- 使用鼠标侧键或键盘按键
- 确认应用与游戏权限一致
- 避免使用被游戏占用的按键
```

MVP 不做：

```text
自动诊断
自动提权
游戏兼容列表
反作弊检测
```

### 优先级

P0。

### 验收标准

```text
用户按住指定快捷键时，自己满足发送条件。
用户松开快捷键后，停止发送。
房间页面清楚显示当前按键。
房间页面清楚显示按键是否被识别为按下。
用户可以修改按键说话快捷键。
修改后立即生效并保存。
应用在后台、游戏在前台时，按键说话仍应可用。
```

## 12.7 自由说话

### 功能描述

自由说话模式下，用户主动开启后，麦克风持续发送声音，不需要按键。

### 行为规则

```text
自由说话必须由用户主动开启。
自由说话不能作为首次进入房间的默认发送状态。
即使本地保存的语音模式偏好是自由说话，下次进入房间也不能自动开麦。
如果用户上次偏好自由说话，下次进入房间时可提示“上次使用自由说话，点击开启”。
```

从按键说话切换到自由说话时，给轻提示：

```text
自由说话已开启，朋友可以听到你的麦克风声音。
```

不使用强弹窗，避免打扰游戏。

### 切换规则

| 当前状态 | 操作 | 结果 |
| --- | --- | --- |
| 按键说话 | 切换到自由说话 | 麦克风持续发送，直到静音、切回按键说话、离开或断线 |
| 自由说话 | 切换到按键说话 | 停止持续发送，等待按键 |
| 自由说话 | 点击静音 | 立即停止发送，底层模式仍为自由说话 |
| 静音中 | 取消静音 | 按当前语音模式规则工作 |

### 优先级

P0。

### 验收标准

```text
用户可以切换到自由说话模式。
自由说话开启后，麦克风保持发送状态。
用户可以随时静音。
用户切回按键说话后，自由说话自动停止发送。
用户下次进入房间时，不会因上次偏好自由说话而自动开麦。
```

## 12.8 自己静音 / 取消静音

### 功能描述

用户可以手动静音自己，无论当前是按键说话还是自由说话。

MVP 中，“静音”只指静音自己。

MVP 不支持：

```text
本地静音某个成员
单独调节成员音量
房主远程静音成员
```

### 行为规则

静音是覆盖状态，不改变底层语音模式。

```text
语音模式 = 按键说话 / 自由说话
静音状态 = 静音 / 未静音
```

发送声音必须同时满足：

```text
未静音
已连接
设备可用
当前语音模式的发送条件成立
```

具体行为：

```text
按键说话模式下静音：即使按住快捷键也不发送。
自由说话模式下静音：停止持续发送，但底层模式仍是自由说话。
静音期间可以切换语音模式。
取消静音后，按当前语音模式规则工作。
```

示例：

```text
自由说话 → 静音 → 切换到按键说话 → 取消静音
结果：回到按键说话，等待按键，不会自动开麦。
```

### 优先级

P0。

### 验收标准

```text
用户可以一键静音自己。
用户可以一键取消静音自己。
静音状态在页面中明显可见。
静音状态不会因为切换窗口而丢失。
静音优先级高于按键说话和自由说话。
```

## 12.9 麦克风设备选择与测试

### 功能描述

用户可以选择用于发送语音的麦克风设备，并通过输入音量条确认是否有输入。

### 设备选择规则

```text
默认麦克风设备 = 跟随系统默认麦克风。
用户可以指定某个麦克风设备。
用户选择后立即用于语音发送。
应用重启后保留选择。
如果指定麦克风不可用，自动回退到系统默认麦克风。
回退时给轻提示：上次选择的麦克风不可用，已使用系统默认麦克风。
如果系统默认麦克风也不可用，显示“未检测到可用麦克风”。
```

### 测试麦克风

MVP 测试麦克风只测试输入音量，不做本地回放。

设置页提供：

```text
当前麦克风设备
麦克风设备列表
输入音量条
无输入提示
```

测试规则：

```text
用户说话时，输入音量条实时变化。
长时间无输入时显示“未检测到麦克风输入”。
不录音。
不回放。
不上传诊断信息。
不判断音质好坏。
```

### 麦克风不可用时进入房间

```text
用户仍可进入房间。
以“仅收听 / 已静音”状态进入。
不发送声音。
自己状态显示：麦克风不可用。
语音模式控制提示需先选择可用麦克风。
用户可以打开设置选择麦克风。
麦克风恢复后，不自动开麦，仍需用户手动取消静音或开启语音。
```

其他成员看到：

```text
静音 / 未发声
不显示具体麦克风异常
```

### 优先级

P0。

### 验收标准

```text
用户可以看到当前麦克风设备。
用户可以切换麦克风设备。
切换后立即用于语音发送。
用户说话时可以看到输入音量变化。
用户下次打开应用时保留上次选择。
指定设备不可用时，应用回退系统默认并提示。
麦克风不可用时，用户仍可进入房间仅收听。
```

默认输出设备 = 跟随系统默认输出设备。
用户可以指定某个输出设备。
用户选择后立即用于房间语音播放。
应用重启后保留选择。

输出设备测试音

每个房间单独保存输出设备

优先回退系统默认输出设备。

应用重启后保留整体输出音量设置。

## 12.11 连接状态与重连

### 功能描述

当网络波动或语音连接异常时，应用应给用户清楚反馈，并尽量自动恢复。

### 状态类型

```text
连接中
已连接
重连中
连接失败
已断开
```

### 首次进入房间连接

```text
输入有效邀请码后进入房间页。
房间页显示语音连接状态：连接中。
连接成功后显示：已连接。
连接失败后显示可操作错误：重试 / 离开房间。
```

连接中时：

```text
不应让用户误以为自己已经能说话。
不发送语音。
明确显示“正在连接语音...”。
```

进入房间超时：

```text
进入房间连接超时为 15 秒。
15 秒内成功则进入已连接状态。
超过 15 秒仍未成功则显示连接失败。
连接失败提供“重试”和“返回首页”。
```

### 自动重连

```text
语音连接异常后自动进入“重连中”。
自动重连最多持续 30 秒。
重连中必须停止发送语音，不能让用户误以为朋友还能听见。
30 秒内重连成功则回到房间。
30 秒内失败则进入“连接失败”。
连接失败后用户可以手动重试或离开房间。
```

### 其他成员视角

```text
断线重连中：成员暂时保留在列表中。
状态显示“重连中”或“连接异常”。
重连成功：恢复在线。
30 秒失败：对其他成员显示短提示“某某已断开”，随后从在线成员列表移除。
```

### 不做内容

```text
复杂重连倒计时展示
网络质量评分
延迟 / 丢包指标
多线路切换
离线消息
语音补听
```

### 优先级

P0。

### 验收标准

```text
连接异常时用户能看到明确状态。
重连中不发送语音。
重连中不让用户误以为自己还能正常说话。
重连失败时提供重试按钮。
重连失败时用户可以离开房间。
其他成员能看到短暂连接异常状态。
```

## 12.12 离开房间、关闭窗口与退出应用

### 功能描述

MVP 必须明确区分“离开房间”“关闭窗口”“退出应用”“异常断开”。

### 状态转换

| 操作 | 行为 |
| --- | --- |
| 点击房间内“离开房间” | 弹出轻确认，确认后离开当前房间，停止发送和接收房间语音，返回首页 |
| 点击窗口右上角 X | 不离开房间，不退出应用，最小化到系统托盘，语音连接继续保持 |
| 托盘菜单点击“退出应用” | 退出前自动离开当前房间，停止语音，关闭应用 |
| 应用崩溃 / 电脑关机 / 进程被杀 | 视为异常断开，其他成员短时间看到连接异常，超过重连窗口后从成员列表移除 |

用户心智：

```text
离开房间 = 不在语音里
关闭窗口 = 仍在语音里，只是隐藏窗口
退出应用 = 离开语音并关闭程序
```

### 离开确认

用户点击离开房间时，应进行轻确认：

```text
确定要离开当前语音房间吗？
[取消] [离开]
```

### 关闭窗口行为

MVP 不提供关闭窗口行为设置。

```text
点击窗口 X 固定最小化到系统托盘。
用户只能通过托盘菜单退出应用。
```

### 优先级

P0。

### 验收标准

```text
用户点击离开房间后，经确认可返回首页。
用户点击窗口关闭按钮后，应用进入托盘并保持语音连接。
用户从托盘退出应用时，自动离开房间并关闭应用。
异常断开时，其他成员最终看到该用户断开并从列表移除。
```

## 12.13 系统托盘

### 功能描述

应用关闭主窗口后，可以继续在后台运行，并在系统托盘中提供基础入口。

### 托盘菜单

MVP 托盘菜单包含：

```text
显示主窗口
当前状态：未在房间 / 房间：{房间名}
离开房间（仅在房间内显示或可用）
退出应用
```

MVP 托盘菜单不包含：

```text
切换静音
切换语音模式
修改音量
当前按键
复制邀请码
```

### 行为规则

```text
点击“显示主窗口”：打开或聚焦主窗口。
点击“离开房间”：弹出轻确认或显示主窗口后确认，确认后离开房间。
点击“退出应用”：如果正在房间内，自动离开房间，然后退出应用。
```

### 优先级

P0。

### 验收标准

```text
用户关闭窗口后，语音不会意外中断。
用户可以从托盘重新打开窗口。
用户可以从托盘离开房间。
用户可以从托盘退出应用。
```

## 15. 关键交互规则

### 15.1 默认进入房间状态

```text
默认语音模式：按键说话
默认快捷键：V
默认不持续发送麦克风声音
默认显示当前语音模式
默认显示当前快捷键
默认显示连接状态
```

### 15.2 语音发送条件

发送声音必须同时满足：

```text
用户未静音
语音连接已连接
麦克风设备可用
当前语音模式发送条件成立
```

按键说话发送条件：

```text
用户按住当前快捷键
```

自由说话发送条件：

```text
用户主动开启自由说话
```

### 15.3 网络动作加载状态

所有网络动作必须有加载中状态，并防重复提交。

适用动作：

```text
创建房间
加入房间
进入房间
重试连接
```

规则：

```text
点击后按钮显示加载中。
禁用重复点击。
成功后进入下一步。
失败后恢复按钮并显示可操作错误。
```

文案示例：

```text
创建中...
加入中...
进入房间中...
正在重试...
```

### 15.4 阻塞错误必须可操作

凡是阻塞用户继续使用的错误，都必须提供至少一个明确下一步操作。

示例：

```text
连接失败：重试 / 返回首页 / 反馈问题
邀请码无效：重新输入 / 返回首页
房间已过期：返回首页 / 创建新房间
房间人数已满：返回首页 / 重试
麦克风不可用：打开设置 / 以仅收听状态进入
输出设备不可用：打开设置 / 继续进入房间
```

## 17. 体验原则

### 17.1 轻

产品体验上应尽量避免：

```text
复杂首页
广告
信息流
推荐内容
大型社区入口
过多弹窗
强制登录
强制绑定手机号
```

### 17.2 快

用户应能在很短路径内完成：

```text
创建房间
复制邀请码
加入房间
开始说话
```

### 17.3 稳

语音状态必须清楚，不能让用户疑惑：

```text
我现在到底有没有开麦？
朋友能不能听见我？
我是不是断线了？
按键说话有没有生效？
麦克风有没有输入？
我能不能听到朋友？
```

### 17.4 少打扰

游戏过程中，应用不应频繁弹窗。

MVP 不做：

```text
提示音
Windows 桌面通知
系统级弹窗
```

重要状态通过以下方式反馈：

```text
状态文字
Toast
状态条
托盘入口
语音状态变化
成员状态变化
```

## 18. 隐私与安全边界

### 18.1 MVP 隐私原则

```text
不要求注册账号。
不要求手机号。
不提供好友关系。
昵称、随机头像、设备偏好保存在本地。
房间语音仅用于实时通信。
MVP 不提供语音录制。
MVP 不保存语音历史。
MVP 不自动上传诊断信息。
```

展示位置：

```text
首次昵称页：简短说明。
关于弹窗：更完整说明或入口。
```

### 18.2 录音与回放

MVP 不支持：

```text
录制房间语音
保存语音历史
回放语音
断线后补听语音
上传语音文件
```

### 18.3 治理功能

MVP 不支持：

```text
举报用户
封禁用户
黑名单
踢出成员
撤销邀请码
房间密码
```

如果邀请码泄露或陌生人进入：

```text
用户可以离开当前房间。
重新创建新的临时房间。
分享新的邀请码给朋友。
```

后续可考虑：

```text
P1 / v0.3：房主踢人、撤销邀请码、关闭房间
P2：房间密码、黑名单、举报机制
```

## 21. 风险与产品应对

| 风险 | 表现 | MVP 应对 |
| --- | --- | --- |
| 用户不知道自己是否开麦 | 误以为朋友能听到或听不到 | 房间页明确展示语音状态、按键状态和输入反馈 |
| 按键说话在游戏中不生效 | 用户体验崩溃 | 后台按键作为 P0；设置页提供兼容性提示；房间页显示按键状态 |
| 麦克风设备选错 | 没声音 | 提供麦克风设备选择、输入音量条、测试麦克风 |
| 输出设备选错 | 听不到朋友 | 提供输出设备选择和整体房间语音音量 |
| 软件关闭后语音中断 | 用户以为只是关窗口 | 关闭窗口固定最小化到托盘 |
| 房间被陌生人加入 | 邀请码外泄 | MVP 通过重新创建临时房间规避；后续支持撤销邀请码和房主踢人 |
| 功能越做越重 | 违背产品初衷 | 坚持 MVP 不做账号、社区、聊天、固定房间等 |
| 隐私担忧 | 用户担心录音或上传 | 明确不录音、不保存语音、不自动上传诊断信息 |

# 轻量级游戏语音工具上下文

本上下文定义这款面向 PC 游戏玩家的轻量多人语音工具中的核心业务语言，避免 PRD 和后续设计中混用房间、会话、邀请码等概念。

## Language

**临时房间**：
一次朋友开黑会话使用的语音房间，通过邀请码邀请成员加入；有成员在线时保持有效，最后一名成员离开后保留 30 分钟，超过 30 分钟无人重新加入则失效。
_Avoid_: 固定房间、长期房间、服务器、频道

**固定房间**：
为固定小队长期保留、可反复进入的语音房间概念；不要用它描述 MVP 的一次性开黑房间。
_Avoid_: 临时房间、当天房间

**邀请码**：
朋友加入某个临时房间时使用的 6 位大写字母数字短码；用户输入时大小写不敏感，并忽略空格和短横线。
_Avoid_: 房间密码、好友邀请、服务器链接

**匿名身份**：
应用在当前电脑上生成的非账号身份，用于在无需注册的前提下识别本机用户和支持重连；它不是手机号、账号或好友关系。
_Avoid_: 账号、用户账号、登录身份

**昵称**：
用户在房间成员列表中展示的名称；MVP 中昵称不是唯一身份，同一房间允许出现重复昵称。
_Avoid_: 用户名、账号名、唯一 ID

**随机头像**：
为本机匿名身份生成并保存的轻量头像，用于房间成员列表展示；MVP 允许重新随机头像，但不支持上传图片、头像库选择或编辑裁剪。
_Avoid_: 上传头像、账号头像、自定义头像

**房主**：
创建临时房间的用户；在 MVP 语境中，房主只是房间内的身份标识，不代表踢人、撤销邀请码、转让房间等管理权。
_Avoid_: 管理员、群主、服务器管理员

**房主管理**：
房主对成员或房间入口进行控制的能力，例如踢出成员、撤销邀请码、关闭房间或转让房主；它属于后续待办，不属于 MVP 房主定义。
_Avoid_: 房主标识、成员列表

**系统托盘**：
Windows 通知区域中的后台入口；MVP 中用于关闭主窗口后保持应用与语音连接运行，并提供显示主窗口和退出应用入口。
_Avoid_: 最小化到任务栏、后台进程、关闭应用

**麦克风设备**：
用户用于发送语音的输入设备；MVP 默认跟随系统默认麦克风，也允许用户指定设备并保留选择，指定设备不可用时回退到系统默认麦克风。
_Avoid_: 音频设备、输入音量、麦克风测试

**输出设备**：
用户用于听取房间语音的播放设备；MVP 默认跟随系统默认输出设备，也允许用户指定设备并保留选择，指定设备不可用时回退到系统默认输出设备。
_Avoid_: 系统音量、游戏声音、整体输出音量

**本地设置**：
不依赖账号、保存在当前电脑上的个人偏好；MVP 保存昵称、随机头像、按键说话快捷键、麦克风设备、输出设备、语音模式偏好和整体输出音量，但不保存关闭窗口行为、房间历史或自动加入上次房间；若保存的语音模式偏好是自由说话，下次进入房间也不能自动发送声音。
_Avoid_: 账号资料、云端配置、房间历史

**语音模式**：
用户在房间内控制麦克风发送方式的选择；MVP 的语音模式只有按键说话和自由说话两种，首次进入房间默认使用按键说话。
_Avoid_: 开麦状态、麦克风权限

**按键说话**：
一种语音模式，用户按住指定快捷键时才发送麦克风声音，松开后停止发送；MVP 默认快捷键为 V，并允许用户修改。
_Avoid_: 对讲、Push-to-talk、PTT

**自由说话**：
一种语音模式，用户不需要按键即可持续发送麦克风声音；它必须由用户在房间内主动开启，不能作为首次进入房间的默认发送状态，也不能因保存偏好而自动开麦。
_Avoid_: 自动开麦、常开麦

## Flagged ambiguities

- “房间”容易同时指临时房间和固定房间；当讨论 MVP 时，默认使用“临时房间”。如果需要长期保留和反复进入，必须明确说“固定房间”。

## Example dialogue

产品：MVP 里用户创建的是临时房间，发邀请码给朋友，当次开黑用。

开发：那第二天输入昨天的邀请码还能进吗？

产品：不承诺。第二天如果还要开黑，重新创建一个临时房间并发送新的邀请码。

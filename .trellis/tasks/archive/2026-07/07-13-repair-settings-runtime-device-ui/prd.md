# 修复设置运行时、设备选择与窗口布局

## 目标

让 Windows echo 桌面端的本地设置页面能够显示真实可用的麦克风与输出设备、在受约束窗口内无页面级垂直溢出，并在 Wails 开发/生产运行时可靠保存设置。

## 已确认事实

- 用户在开发模式观察到 Save 后请求 `http://wails.localhost:9245/wails/custom.js` 返回 404；RPC 请求载荷和设置返回 JSON 均可见。`@wailsio/runtime` 将该脚本定义为可选 server-mode 注入，缺失时静默忽略，不能作为 Save 失败证据。
- 当前 `App.tsx` 将麦克风和输出设备渲染为自由文本输入；它不枚举系统设备。
- 当前 Wails 窗口只设置 `Width: 1000`、`Height: 618` 和深色 native 背景，未设置最小/最大尺寸。
- ADR 0007 规定媒体设备枚举、采集、播放和切换归 WebView2 浏览器 API 与 LiveKit JS；Go/Wails 只保存偏好和提供桌面壳。输出设备若 WebView2 不稳定，必须回退到系统默认输出并清楚提示。
- 当前 Vite 使用 Wails runtime Vite plugin，监听 `127.0.0.1:${WAILS_VITE_PORT:-9245}`；保存失败必须以 binding Promise、返回设置和持久化结果判断，而不是以可选 `custom.js` 网络请求判断。

## 需求

### R1：真实设备来源与选择

设置页面必须从 WebView2 的媒体设备 API 获取输入/输出设备，不得让用户手填设备 ID。设备标签不可用时须请求或解释麦克风权限；设备缺失、权限拒绝或输出切换不受支持时提供可操作的回退文案。选中的设备 ID 继续通过现有本地设置服务保存。

### R2：窗口约束与无溢出布局

Wails 主窗口须声明符合当前设置页的最小/最大尺寸约束。设置页面在支持的最小尺寸和默认尺寸下不得出现页面级纵向滚动条；内容过多时只允许卡片内部可访问地滚动。WebView 和 native 窗口背景须使用同一浅色画布。

### R3：可靠保存运行时链路

在真实 Wails desktop dev 运行时，Load、Save 和 ResetAvatar 的 binding Promise 必须正确结算；Save 成功后 UI 显示明确成功状态并采用服务返回值，重启后恢复保存的设备、音量、快捷键、昵称和语音偏好；调用失败时保留可操作错误与重试。缺失的可选 `/wails/custom.js` 不得被误判为保存失败，也不得通过空 no-op 资源掩盖。

### R4：保留语音安全边界

保存 `free_talk` 偏好仍不得打开房间内自由说话；现有连接、麦克风可用、静音和显式 room enablement 门控不变。

## 验收标准

- [ ] AC1：获得媒体权限后，设备控件显示实际枚举到的输入/输出设备及“跟随系统默认”选项；用户无法手填任意设备 ID。
- [ ] AC2：权限拒绝、无可用设备及不支持输出设备切换时显示中文可操作状态，不伪造可用设备或已切换成功。
- [ ] AC3：选择设备并保存后，服务收到对应 device ID；重新启动后恢复偏好。设备不存在时回退系统默认且提示。
- [ ] AC4：在 Wails 默认窗口及声明的最小尺寸下，设置页面没有 viewport 级纵向滚动；内容与主操作仍可访问。
- [ ] AC5：真实 `wails3 dev` desktop 运行 Save 后显示成功状态、更新 UI 且重启恢复；Load、Save、ResetAvatar 的实际 binding rejection 显示错误和重试。可选 `custom.js` 404 仅作为 Wails runtime 行为记录，不作为 RPC 失败。
- [ ] AC6：现有昵称 Unicode 校验、损坏配置安全回退、随机头像、`free_talk` 房间门控及本地设置范围均不回归。

## 非目标

- 新建 Go 音频采集、编码、播放、混音或设备驱动实现。
- 已加入 LiveKit 房间后的完整音频切换实现以外的房间、账号、云同步、历史或自动加入功能。
- 通过管理员权限、反作弊规避或任意 native hook 修改来强制设备访问。

## 未决问题

- 无。ADR 0007 已定义输出设备不受支持时的系统默认回退策略；实现前仍须以 Wails/WebView2 spike 验证实际行为。

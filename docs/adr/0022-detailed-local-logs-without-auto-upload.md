# Keep detailed local logs without automatic upload

Echo MVP will keep detailed structured logs on both the client machine and the server machine because WebRTC, device selection, global push-to-talk, room lifecycle, and reconnect failures are otherwise difficult to debug. Client logs should cover application lifecycle, Wails shell events, tray behavior, key registration and press/release state, device enumeration and switching, LiveKit state, WebSocket state, room operation failures, and uncaught errors; server logs should cover HTTP request summaries, room lifecycle events, invite validation, capacity checks, WebSocket connections, reconnect windows, LiveKit token issuance results, LiveKit API calls, and database errors. Logs must not record voice content, audio data, long-lived plaintext invite history, plaintext LiveKit tokens, or sensitive request bodies, and MVP will not automatically upload logs; troubleshooting uses manual export or copy/paste only.

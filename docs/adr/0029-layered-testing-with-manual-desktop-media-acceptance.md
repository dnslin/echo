# Use layered tests with manual desktop media acceptance

Echo will use automated tests for deterministic business and UI state rules, and manual acceptance for Windows desktop media behavior that depends on WebView2, LiveKit, audio devices, system tray behavior, and foreground games. Backend tests should cover invite normalization, room lifecycle, capacity, reconnect windows, retention expiry, token issuance, and SQLite persistence; frontend/client tests should cover local settings, voice mode state, mute priority, push-to-talk pressed/released state, WebSocket state handling, and key UI status rendering. Full automated end-to-end coverage of LiveKit audio, device switching, and global game foreground push-to-talk is deferred because those paths need real Windows environment validation for MVP.

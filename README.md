# echo

面向 PC 游戏玩家的轻量多人语音工具。目标是让 2-10 名朋友在 Windows 10 / Windows 11 x64 上无需注册，通过 6 位邀请码快速进入同一个临时房间，并完成稳定的游戏语音交流。

当前阶段：v0.1 / MVP / 朋友内测版。

## 产品边界

echo 优先解决一件事：朋友开黑时快速进房、能说话、能听见、状态清楚、低打扰。

MVP 包含：

- 本地匿名身份、昵称和随机头像
- 创建临时房间并生成邀请码
- 输入邀请码加入临时房间
- 2-10 人实时语音
- 按键说话和自由说话
- 自己静音 / 取消静音
- 成员列表、正在说话、静音、重连状态
- 麦克风设备和输出设备选择
- 测试麦克风输入和整体房间语音音量
- 关闭窗口进入系统托盘，语音不中断
- 网络异常时自动重连并给出清晰状态
- 朋友内测安装包通过 GitHub Releases 分发

MVP 不包含：账号、好友、固定房间、文字聊天、视频、录音、房主管理、举报封禁、自动更新、移动端、网页端完整体验、Overlay、开机启动、语音激活、单人成员音量。

## 关键术语

- 临时房间：一次朋友开黑会话使用的语音房间；最后一名成员离开后保留 30 分钟。
- 邀请码：朋友加入临时房间时使用的 6 位大写字母数字短码；输入时大小写不敏感，忽略空格和短横线。
- 匿名身份：当前电脑生成的非账号身份，用于无需注册的本机识别和重连。
- 房主：创建临时房间的用户；MVP 中只是成员列表身份标识，不具备踢人、撤销邀请码或关闭房间能力。
- 按键说话：按住指定快捷键时发送声音，松开停止发送。
- 自由说话：用户在房间内主动开启后持续发送声音；不能作为首次进入房间默认状态。

完整术语以 `CONTEXT.md` 为准。

## 目标架构

```text
Windows 客户端
  Wails 3 Go shell：窗口生命周期、系统托盘、按键 hook、本地设置、本地日志
  React / TypeScript / WebView2：房间 UI、设备选择、成员状态、用户交互
  LiveKit JS：WebRTC 音频采集、发布和播放

公网服务
  外部 Nginx：HTTPS / WSS 入口和反向代理
  Gin 业务服务：临时房间、邀请码、成员状态、房间生命周期、短期连接凭证
  SQLite：产品房间生命周期持久化
  LiveKit：SFU 媒体转发
```

详细设计以 `design.md` 为准。

## 实施入口

实现时从 `implement.md` 开始，并按任务顺序执行。

执行约束：

1. 先完成 Wails 3 + WebView2 + LiveKit、设备/托盘、按键说话三个技术 spike。
2. spike 通过前，不开始完整功能实现。
3. 每个任务按 `implement.md` 中的验证命令通过后再提交。
4. 不引入账号、聊天、固定房间、自动更新、TURN、Redis、PostgreSQL、Electron 或服务端混音。
5. `prd.md`、`CONTEXT.md`、`design.md` 和 `docs/adr/` 是上下文来源，不在代码中重新发明产品语义。

## 验收方向

MVP 是否可以进入朋友内测，至少需要覆盖：

- Windows 10 / Windows 11 x64 安装、启动和退出
- 两人进房并双向语音
- 3-10 人房间语音
- 第 11 人加入失败并看到明确提示
- 麦克风设备切换、输出设备切换、测试麦克风输入
- 游戏前台时按键说话连续按下 / 松开
- 自由说话必须由用户在房间内主动开启
- 静音阻止发声
- 关闭窗口进入系统托盘后语音不中断
- 30 秒内重连恢复，超过 30 秒进入连接失败
- 空房 30 分钟后房间过期
- GitHub Releases 安装包可安装并启动

验收标准以 `prd.md` 为准；具体手工验收记录应放入后续独立验收清单。

## 决策索引

重要决策已记录在 `docs/adr/`，包括：

- 服务端转发语音，而不是纯 P2P
- SFU 风格媒体转发，而不是服务端混音
- WebRTC / LiveKit 承载语音媒体
- Wails 3 桌面壳，Electron 不是回退方案
- Windows native keyboard hook 作为按键说话可靠性路径
- Gin + GORM + SQLite 业务服务
- 单实例 MVP 部署
- 客户端生成匿名身份
- 服务端生成邀请码和短期连接凭证
- 30 秒重连保留成员
- 30 分钟空房生命周期
- Docker Compose + 外部 Nginx 公网部署
- 分层自动化测试 + Windows 桌面媒体手工验收

需要理解“为什么这么做”时，先读对应 ADR，再改设计或实现。
